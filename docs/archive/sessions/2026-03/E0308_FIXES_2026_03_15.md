# E0308 Systematic Fixes - 2026-03-15

## Summary

Implemented compiler fixes to eliminate E0308 "mismatched types" errors across 230 errors in phase5_clean_final.txt.

## Fixes Implemented

### 1. Float Inference: Unary Deref for Match Arm Unification (Category E)

**File:** `windjammer/src/type_inference/float_inference.rs`

**Problem:** Match arms like `Some(v) => *v, None => 999999.0` - the `*v` yields f32 (from HashMap.get), but the literal 999999.0 defaulted to f64. MustMatch constraint existed but *v never got a type to propagate.

**Fix:** 
- Added `Expression::Unary` handling in `collect_expression_constraints` to recurse into operands
- When *v where v: &f32, add MustBeF32 constraint for the deref expression
- Enables constraint solver to unify: *v (f32) ↔ 999999.0 (now gets f32)

**Impact:** Fixes ~2 match arm f32/f64 errors (astar_grid.rs)

### 2. Index Deref for Copy Type Parameters (Category D)

**File:** `windjammer/src/codegen/rust/expression_generation.rs`

**Problem:** 
- `sdf_sphere(x, y, z, node.params[0], ...)` - In Rust, `vec[i]` returns `&T`. Function expects `f32`.
- `update_bone_recursive(roots[j], ...)` - Same: `roots[j]` gives `&u32`, method expects `u32`.

**Fix:**
- **Call handler:** When param expects Copy type (f32, u32, i32, etc.) and arg is `Expression::Index`, wrap in `*(arg_str)`
- **MethodCall handler:** Same logic for method arguments

**Impact:** Fixes ~12+ "expected f32, found &f32" errors (csg/evaluator.rs, animation/skeleton.rs)
