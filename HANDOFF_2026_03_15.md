## 🎨 Shader Ergonomics: Priority 1

### The Problem (Colored Stripe Debugging)

**Current painful flow**:
1. Write WGSL shader
2. Run game → **Colored stripes!** 😢
3. Debug cryptic error: "validation error"
4. Guess what's wrong
5. Recompile game (30+ seconds)
6. Repeat 10+ times
7. Finally works after 2+ hours

**Root causes**:
- Raw WGSL with manual buffer offsets
- No compile-time type checking
- Poor error messages
- Can't TDD shaders
- Slow iteration cycle

### The Solution: Windjammer Shader Language (.wjsl)

**New flow**:
1. Write .wjsl shader (type-safe)
2. Compile → **Type error at line 15!** (clear)
3. Fix typo
4. Test shader (<1 second with TDD)
5. Run game → **Perfect rendering!** ✅

**Features**:
- ✅ Type-safe shader definitions
- ✅ Compile-time validation
- ✅ Clear error messages with line numbers
- ✅ TDD-testable shaders
- ✅ <1 minute iteration time
- ✅ Automatic buffer layout
- ✅ Binding group management

**Example**:
```windjammer
// shader/pbr_lighting.wjsl
@vertex
fn vs_main(in: VertexInput) -> VertexOutput {
    let world_pos = model_matrix * vec4(in.position, 1.0)
    let clip_pos = view_proj * world_pos
    
    return VertexOutput {
        position: clip_pos,
        world_pos: world_pos.xyz,
        normal: (model_matrix * vec4(in.normal, 0.0)).xyz,
        uv: in.uv,
    }
}

@fragment  
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    let albedo = texture_sample(albedo_texture, in.uv)
    let normal = normalize(in.normal)
    let light_dir = normalize(light_position - in.world_pos)
    
    let diffuse = max(dot(normal, light_dir), 0.0)
    return vec4(albedo.rgb * diffuse, 1.0)
}
```

**Benefits**:
- **10x faster** shader development
- **Zero colored stripe debugging**
- **Type errors caught at compile time**
- **TDD-testable** (test shaders like regular code)
- **Clear error messages** ("expected vec3, got vec4" not "validation error")

**Timeline**: 2-3 weeks to production
